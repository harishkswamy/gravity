Dependency Injection
Types of Injection

Software components seldom operate in isolation; they often collaborate with other components to form a cohesive whole. This collaboration leads to dependencies between components that is often times a hassle to manage.

Example 1.1. Conventional Dependency Management

class ComponentA
{
    void doSomething()
    {
        ...
        ComponentB componentB = componentBFactory.getComponentB();
        componentB.doSomething();
        ...
        ComponentC componentC = componentCFactory.getComponentC();
        componentC.doSomething();
        ...
    }
}

This obscures the business logic in code and leads to a proliferation of factories.

Dependency Injection

Dependency Injection / Invertion of Control is the design pattern employed to free the components from the hassle of managing their dependencies. Instead, the dependencies are configured and injected into the component by an external entity - the container.

Example 1.2. Dependency Management via Injection

class ComponentA
{
    ComponentB _componentB;
    ComponentC _componentC;
    
    void doSomething()
    {
        ...
        _componentB.doSomething();
        ...
        _componentC.doSomething();
        ...
    }
}

Here the dependencies are already there when the component needs it; hence the code is clear and there are no factories littered thought the system.

Types of Injection

Constructor Injection

Constructor Injection is a technique used to inject component dependencies via the component's constructor.

Method Injection

Method Injection is a technique used to inject component dependencies via the component's public methods.

Combo Injection

Combo Injection is a combination of constructor and method injections. Combo Injection is a technique used to inject component dependencies via both the component's constructor and its methods.
